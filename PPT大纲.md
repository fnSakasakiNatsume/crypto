## 第4页：核心模块1 - 密钥交换（1分钟）
**标题：** RSA密钥交换演示

**数学原理：**
- **加密公式：** `C ≡ PMS^e (mod n)`
- **解密公式：** `PMS ≡ C^d (mod n)`
- **安全性：** 只有拥有私钥d的服务器才能解密

**演示视频流程示意：**
1. **服务器生成RSA密钥对**（2048位）
   - 显示：公钥/私钥生成过程
   - 代码：`KeyPairGenerator.getInstance("RSA")`

2. **客户端生成Pre-Master Secret**
   - 显示：48字节随机数生成
   - 说明：包含TLS版本号（前2字节）+ 随机数据（46字节）

3. **客户端加密PMS**
   - 显示：使用服务器公钥加密PMS
   - 输出：加密后的密文（Base64编码）

4. **服务器解密PMS**
   - 显示：使用私钥解密，恢复原始PMS
   - 验证：解密结果与原始PMS完全匹配

5. **生成会话密钥**
   - 公式：`Session Key = SHA-256(R1 || R2 || PMS)`
   - 说明：双方使用相同的R1、R2、PMS，生成相同的密钥

**关键点：**
- ✓ 非对称加密用于安全交换对称密钥
- ✓ 只有服务器能解密PMS，保证安全性

## 第5页：核心模块2 - 证书验证（1分钟）
**标题：** 数字证书与CA签名验证

**核心机制：**
- **CA签名：** `Signature = RSA_Encrypt(SHA-256(Certificate), CA_PrivateKey)`
- **客户端验证：** `Verify(Signature, Hash(Certificate), CA_PublicKey)`

**演示视频流程示意：**
1. **CA生成密钥对**
   - 显示：CA密钥对生成（RSA 2048位）
   - 说明：CA私钥用于签名（绝密），CA公钥用于验证（公开）

2. **CA对证书签名**
   - 步骤1：计算证书哈希值（SHA-256）
   - 步骤2：使用CA私钥对哈希值签名
   - 输出：数字签名（Base64编码）

3. **客户端验证签名**
   - 步骤1：重新计算证书哈希值
   - 步骤2：使用CA公钥验证签名
   - 结果：✓ 验证通过 → 服务器身份可信

4. **中间人攻击防护演示**
   - 场景1：真实CA签名 → 验证通过 ✓
   - 场景2：攻击者伪造签名 → 验证失败 ✗
   - 场景3：证书被篡改 → 验证失败 ✗

**关键点：**
- ✓ 防止中间人攻击（MITM）
- ✓ 攻击者无法伪造CA签名（没有CA私钥）
- ✓ 证书篡改会被检测到（哈希值不匹配）

## 第6页：核心模块3 - 对称加密（45秒）
**标题：** AES-GCM对称加密通信

**算法特性：**
- **算法：** AES-256-GCM
- **模式：** Galois/Counter Mode
- **优势：** 提供加密和认证（Authenticated Encryption）

**演示视频流程示意：**
1. **从会话密钥派生AES密钥**
   - 显示：从会话密钥提取32字节作为AES密钥
   - 说明：实际TLS中会分割用于不同用途

2. **客户端加密HTTP报文**
   - 输入：HTTP请求报文（明文）
   - 过程：
     - 生成随机IV（12字节）
     - AES-GCM加密
   - 输出：密文 + IV（Base64编码）

3. **服务器解密HTTP报文**
   - 输入：密文 + IV
   - 过程：AES-GCM解密
   - 输出：原始HTTP报文（明文）

4. **篡改检测演示**
   - 场景：修改密文1个字节
   - 结果：解密失败，抛出异常
   - 说明：GCM模式自动验证数据完整性

**关键点：**
- ✓ 对称加密比非对称加密快得多
- ✓ GCM模式提供加密和认证双重保护
- ✓ 每次加密使用不同IV，相同明文产生不同密文

## 第8页：安全性分析（45秒）
**标题：** 安全性保障机制

**多层安全防护：**

1. **密钥交换安全性**
   - RSA加密：基于大数分解难题
   - 只有服务器能解密PMS（拥有私钥）
   - 即使攻击者截获密文，也无法恢复PMS

2. **身份验证安全性**
   - CA签名：防止证书伪造
   - 攻击者无法获得CA私钥
   - 证书篡改会被检测到（哈希验证）

3. **数据加密安全性**
   - AES-256：256位密钥，计算安全
   - GCM模式：提供加密和认证
   - 每次使用不同IV，防止重放攻击

**局限性说明：**
- ⚠️ 本项目演示RSA密钥交换（TLS 1.2方式）
- ⚠️ TLS 1.3已移除RSA密钥交换，改用ECDHE
- ⚠️ ECDHE提供前向安全性（Forward Secrecy）

**扩展话题：**
- TLS 1.3改进：0-RTT握手、更快的速度
- 前向安全性：即使服务器私钥泄露，历史通信仍安全
